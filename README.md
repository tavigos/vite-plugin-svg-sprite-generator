# vite-plugin-svg-sprite-generator

🌐 English | 🇷🇺 [Русский](./README.ru.md)

A modern, high-performance, and **completely zero-dependency** SVG sprite generator for Vite 5, 6, 7, and 8+. 

It leverages native Node.js capabilities for recursive directory traversal, ensuring maximum speed, minimal package size, and zero vulnerability risks in your dependency chain.

## 🚀 Features

* **Zero Dependencies** — Does not pull megabytes of data into your `node_modules` (no `fast-glob`, `cheerio`, or `svgo`). Runs strictly on native Node.js APIs.
* **Two Build Modes** — Combine all icons into a single global file or automatically split them into separate sprites based on your folder structure.
* **Draft Support** — Files starting with an underscore `_` are automatically ignored.
* **Smart Attribute Management** — Flexible attribute transfer from `<svg>` to `<symbol>` with the ability to locally force inheritance using the `!` prefix.
* **Live Watch Reloading** — Instantly regenerates sprites when files are added, changed, or deleted during development.
* **Safe Cleaning** — The smart `clean` algorithm deletes only the files generated by the current build, preventing accidental deletion of user-defined SVGs (like `logo.svg` or `favicon.svg`).

## 📦 Installation

For stable operation, the plugin requires **Node.js >= 18.5.0** (where native recursive directory reading was introduced) and **Vite >= 5.0.0**.

```bash
npm install vite-plugin-svg-sprite-generator --save-dev
# or
pnpm add -D vite-plugin-svg-sprite-generator
# or
yarn add -D vite-plugin-svg-sprite-generator

```

## 🛠 Usage

Add the plugin to your `vite.config.js` configuration file:

```javascript
import { defineConfig } from 'vite';
import svgSprite from 'vite-plugin-svg-sprite-generator';

export default defineConfig({
    plugins: [
        svgSprite({
            input: 'src/icons',
            output: 'dist/assets',
            mode: 'single',
            clean: true
        })
    ]
});

```

## ⚙️ Configuration Options (API)

All options are optional.

| Option | Type | Default | Description |
| --- | --- | --- | --- |
| `input` | `string` | `'src/icons'` | Path to the source SVG folder (scanned recursively). |
| `output` | `string` | `'dist/assets'` | Path to the directory where generated sprites will be saved. |
| `name` | `string` | `'sprite.svg'` | Base name of the sprite file. The `.svg` extension is optional. |
| `mode` | `'single' | 'folders'` | `'single'` | Icon grouping mode (one global file or split by folders). |
| `clean` | `boolean` | `true` | Clean up old generated sprites before a new build. |
| `inheritSvgAttrs` | `boolean` | `true` | Whether to transfer attributes from the original `<svg>` (except `xmlns` and `id`) to `<symbol>`. |

## 📂 Build Modes (`mode`)

### `single` Mode

All SVG files from the `input` folder (including all subfolders) are compiled into a single global sprite file.

* **Output filename:** `sprite.svg` (or the value from the `name` parameter).

### `folders` Mode

The plugin groups icons by their first subfolder relative to the `input` path.

If your structure looks like this:

```text
src/icons/
├── editor/
│   ├── plus.svg
│   └── minus.svg
├── code/
│   ├── js.svg
│   └── ts.svg
└── flags/
    └── us.svg

```

The following files will be generated in the `output` folder:

* `sprite-editor.svg` (contains `plus` and `minus` icons)
* `sprite-code.svg` (contains `js` and `ts` icons)
* `sprite-flags.svg` (contains `us` icon)

> **Note:** If you set `name: ""`, the `sprite-` prefix is omitted, and the files will be named exactly after the folders: `editor.svg`, `code.svg`, `flags.svg`. Nesting deeper than the first level (e.g., `editor/ui/add.svg`) will still end up in the first-level sprite (`sprite-editor.svg`).

## 🌟 Advanced Features

### Hidden SVGs (Drafts)

If you want to keep working drafts or source icons inside your project folder without publishing them, simply add a leading underscore `_` to the filename. The plugin will completely ignore them:

* `_plus-old.svg` — **ignored**
* `minus.js` — **ignored** (the plugin only processes `.svg` files)

### Attribute Inheritance & Local Overrides

By default (`inheritSvgAttrs: true`), all presentational attributes of the original `<svg>` tag (such as `viewBox`, `fill`, `stroke`, `stroke-width`) are transferred to the `<symbol>` tag. The `xmlns` and `id` attributes are always excluded automatically.

If set to `inheritSvgAttrs: false`, attributes are not inherited, and only the `viewBox` is preserved (if it was originally present).

#### 🔥 Local Override (`!` Prefix)

If a filename starts with an exclamation mark `!`, the plugin **will force attribute inheritance** for that specific icon, regardless of the global `inheritSvgAttrs` setting.

After generation, the `!` prefix is automatically removed from the final symbol `id`:

* File: `!google.svg` ➔ `<symbol id="google" viewBox="..." fill="...">`

## 🪵 Logging & Error Handling

The plugin is designed to never crash the `Vite Build` process due to missing files or empty directories. Instead of throwing errors, it outputs clean, non-blocking warnings.

Upon successful generation of each file, detailed info is printed to the Vite console:

```text
[vite-plugin-svg-sprite-generator]

Created:
sprite-editor.svg
(15 icons)

```
